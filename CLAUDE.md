Rara Calendar 프로젝트를 생성합니다.
/프로젝트명 : 랄라라 캘린더

/폴더명 : rara-calendar

웹앱에서 작동하고 음력표시가 되고, 날짜 한 칸에 3가지 일정(3줄)까지 보이게 하며,
알람기능, 투두 리스트까지 포함된 나만의 ‘랄라라 캘린더’를 만드려고 한다.
전체적인 UI디자인은 참고사진과 같게 하고 싶고 색톤만 그레이 파스텔톤으로 바꿔 만들고 싶다.
내가 생각하는 화면구조를 간략히
설명할테니 잘 확인하고 추가 또는 수정할 부분이 있다면 조언과 추천 부탁해.


1.메인화면 : 달력이 나오는 화면으로 음력날짜도 함께 표기한다.
-참고 사진에 달력이 나오는 사진을 보면 달력에서 날짜 한 칸에 그 날짜에 입력한 일정 중, 첫일정의  맨 앞에 쓴 다섯글자만 보이도록 하고
나머지 일정들은 ‘+숫자’ 처럼 표시해서 일정이 1개가 아니라 여러개가 있다는 것을 표시한다.
-음력날짜를 참고사진처럼 날짜 숫자와 함께 작은 글자로 표현한다. 음력표기 날짜는 한달에 4개로 날짜는 랜덤선택 또는 절기를 고려해서 입력한다.
-입력내용이 있는 날은 날짜 칸의 배경색 아주 연하게 넣어서 시각화 한다. 
-입력란하단부에 날짜 한칸이동키가 좌,우로 표현되게하고 오늘날짜로 이동하는 키(Today)와 
입력내용이 있는 날짜로 이동되었을 때에는 today버튼과 삭제 이미지가 활성화 되면서 각각의 기능을 수행하게 한다. 
그리고 투두 리스트를 보여주는 아이콘도 하나 추가해서 생성하고 이 버튼을 눌렀을때,
 하단 부에서 올라오는 슬라이딩 창에 오늘날짜를 기준해서 10일 이후 일정을 체크 박스 형태로 일정을 보여주게 한다. 
-날짜를 선택할때마다 배경색이 나타나면서 선택한 상태를 보여준다. 
-오늘날짜 기준 표시일 : 캘린더와 입력내용란 사이에 한 줄 라인을 만들고
오늘 날짜를 기준으로 현재 선택된 날짜와의 차이르 계산해서 그 값을 ‘어제’, ‘내일’, 또는 ‘2일 전’, ‘2일 후’ 등으로 표시해 준다.

-우측 상단에 관리자 이모티콘을 만들어서 관리자화면으로 이동하게 한다. 
-년,월의 이동은 좌우로 슬라이딩해서 이동한다. 

-메인화면의 구성 : 상단고정메뉴바를 만들어서
첫줄에는 검색아이콘, 년월 표시, 관리자아이콘을 넣고,
둘째줄에는 요일,
세째줄에는 가늘고 옅은색으로 된 구분선을 넣어준다.

2.캘린더
-일요일부터 토요일까지 차례로 캘린더 구성함.
-일요일에 들어가는 숫자는 빨간색으로 한다.
-오늘 날짜에 해당하는 날짜칸에는 숫자와 함께 오늘을 의미하는 아주 작은 아이콘 또는 ‘오늘’이라는 글자를 주황색으로 넣어준다.

3.오늘날짜 기준 표시일 : 캘린더와 입력내용란 사이에 한 줄 라인을 만들고
오늘 날짜를 기준으로 현재 선택된 날짜와의 차이르 계산해서 그 값을 ‘어제’, ‘내일’, 또는 ‘2일 전’, ‘2일 후’ 등으로 표시해 준다.

4.메인화면에서의 내용입력란
/캘린더에서 입력내용이 있는데 오늘 날짜를 클릭 또는 터치하면 내용입력란에 입력내용을 보여준다.
이때 동시에 입력란하단부메뉴바의 ‘좌,우 이동방향키’, ’Today’버튼, ‘삭제아이콘’ 모두 활성화 된다.
/캘린더에서 입력내용이 있는 날짜를 클릭 또는 터치하면 내용입력란에 입력내용을 보여준다.
이때 동시에 입력란하단부메뉴바의 ‘좌,우 이동방향키’, ’Today’버튼, ‘삭제아이콘’ 모두 활성화 된다.
/캘린더에서 입력내용이 없는 날짜를 클릭 또는 터치하면 내용입력란에 아무 내용도 보이지 않는다.
이때 동시에 입력란하단부메뉴바의 ‘좌,우 이동방향키’, ’Today’버튼은 활성화되고, ‘삭제아이콘’ 비활성화 된다.

/입력란하단부메뉴바 : 캘린더의 년,월 이동시, 내용입력란 맨 하단부의 좌측쪽에 붙은 위치로 고정되어서 보여준다.
/ 예시 : ‘ <     Today    삭제아이콘    >                       Todolist ’
/메인화면 내용입력란의 ‘입력란하단부메뉴바’에는
입력내용이 있는 날짜들 사이에서 이동할 수 있는 ‘좌,우 이동방향카’와 캐린더에서 오늘 날짜로 돌아 갈 수 있는  ’Today’	버튼,
그리고 쓰레기통 모양의 ‘삭제아이콘’이 위 예시 처럼 만든다.
그리고 투두 리스트를 보여주는 ‘아이콘’ 또는 ‘Todolist’ 텍스트로 만든 버튼을 맨우측 끝에 추가해서 생성하고 이 버튼을 눌렀을때,
 하단 부 화면 아래에서 슬라이딩 창이 올라오게 해서,
오늘날짜를 기준해서 10일 이후의 일정들 모두를 체크 박스 형태를 넣어서 일정리스트를 보여주게 한다.


5.검색화면 : 참고사진 반영
-메인화면의 상단고정메뉴바에 있는 ‘검색아이콘’을 클릭했을때 나오는 검색화면.
-
-‘검색아이콘’을 클릭했을때 나오는 첫화면 디자인 : 최상단 메뉴바 (한줄), ‘List’ 텍스트 (한줄), 검색창 (한줄), 입력내용이 있는 날짜들을
기본적으로 최신순으로 목록을 만들어서 보여줌
/ 최상단메뉴바 : 좌측끝에 편집(edit)버튼, 우측끝에 이전화면(back) 
+편집(edit)버튼 클릭 : 입력내용이 있는 목록들이 모두 우측으로 살짝 이동하면서 목록들 맨 앞에 모두 동그라미 모양의 체크 박스를 만들어
삭제기능을 만든다. 이때 전체 선택 버튼도 만들어서 모두 선택해서 일괄 삭제도 할 수 있게 한다. 이때 
전체선택 버튼 ’Select All’ 과 삭제(Delete)버튼은 검색화면에서 편집시에,
 살짝 투명도가 있는 ‘검색편집시 전체선태과삭제메뉴바’ 를 하단에 한줄로 만들어서 고정시킨디.
‘검색편집시 전체선태과삭제메뉴바’의 죄측끝에 전체선택 버튼 ’Select All’을 위치시키고, 우측끝에 삭제(Delete)버튼을 배치한다.
검색화면에서 편집(edit)버튼을 클릭했을때, 그 자리에 취소(Cancel)버튼이 보이게 한다. 편집버튼과 취소버튼은  같은 자리에서 변한다.
입력내용이 있는 목록들은 내용이 들어 있는 라인 부분을 우측에서 좌측으로 끌어서 삭제 가능한 기능도 구현한다.

+이전화면(back)버튼 클릭 : 메인화면으로 이동함.

-리스트검색, 편집(edit), 선택삭제, 전체삭제,취소, 메안화면으로이동(back)

## 🛠 기술 구현 계획

### 기술 스택 (최종 결정)
- **프론트엔드**: React 18 + Vite + TypeScript
- **백엔드**: Node.js + Express.js
- **데이터 저장**: localStorage (클라이언트 측)
- **스타일링**: Tailwind CSS (파스텔 톤 커스터마이징)
- **날짜 라이브러리**: Day.js
- **음력 변환**: lunar-calendar 또는 custom 함수
- **알람**: Web Notifications API + Service Workers (PWA)
- **UI 컴포넌트**: 커스텀 구현 (참고 사진 기반)
